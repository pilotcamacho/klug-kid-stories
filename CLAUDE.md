# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

**Klug Kid Stories** is a language learning web app built around spaced repetition and story-based contextualization. The core premise:

- Vocabulary progress is tracked per **word meaning** (not per word form). Each entry represents one specific semantic sense of a word.
- Words are always stored in their **base/lemma form** (no declination, conjugation, or inflection).
- The app is **language-agnostic** — designed to support any language pair.
- Students are evaluated by **typing answers**, never by self-assessment.

## Core Concepts

### Spaced Repetition

The app implements a spaced repetition system (SRS) to schedule word reviews:

- Each word meaning has a **retention score** — an estimate of how far into the future the student will still remember it.
- Review intervals are computed from a **forgetting curve model** fitted to the individual student's historical recall performance and aggregate data from other users (for cold-start).
- Correct answers push the next review further into the future; incorrect answers shorten the interval.
- The scheduler surfaces word meanings that are approaching the predicted forgetting point.

### Story-Based Contextualization

Words are never asked in isolation. The system presents the student with a **short story containing blanks**:

- Each story is generated using **only words the student has already learned** (words with sufficient retention score), ensuring full comprehension of the surrounding context.
- One or more blanks correspond to the word meanings being tested in that session.
- The student types the missing word(s) to complete the story.
- Answer evaluation accepts valid inflected forms (conjugations, plural, case, etc.) of the correct lemma.

### Word Entry

Students can add vocabulary in three ways:
1. **Pre-loaded** — A shared pool of `WordMeaning` records seeded into DynamoDB by an admin script, sourced from open frequency corpora (e.g. OpenSubtitles, Wiktionary). Seed data lives under `data/seeds/<language-code>/`; loaded by `scripts/seed.ts`.
2. **Manual entry** — the student defines a word meaning directly.
3. **Text import** — the student pastes a text; the system extracts, lemmatizes, and deduplicates the vocabulary.

In all three cases, a `UserWordProgress` record is created the first time the scheduler introduces the word — that moment counts as "new word" for daily limit purposes.
